Bug Discovery and Fix: `ramalama list` Crash (#2586 → PR #2588, Merged)

Type: Bug Report + Code Fix | Status: Merged

What I did: While working on my assigned RamaLama task (#124), I discovered a reproducible regression in ramalama==0.18.0 where ramalama list crashed with AttributeError: 'str' object has no attribute 'timestamp'. I opened issue #2586 with clear reproduction steps and environment details, then submitted PR #2588 to normalize manifest fields for unsupported manifest-attributes engines by parsing modified values and normalizing size.

Impact: The fix was merged upstream and included test coverage, preventing this crash path and improving command reliability for users on affected setups.
